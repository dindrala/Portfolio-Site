Getting started

The package you downloaded should contain the following directories:

documentation - contains the help file and its assets.
Fireworks PNG - Layered png graphic files.
html - contains all the theme files.
PSD - PSD files ( note: they are generated by Fireworks )

Creating a backup folder for the original template files is highly recommended before start the work with them. Those backup files can be very useful in case the development process goes wrong.

You will need to have a text editor installed in order to start the work with the html, js or css files. Personally I prefer Context, but the Notepad++ is also a really good and widely used tool.

The theme comes with 18 pre-made homepage templates. Once you selected one of them as the future landing page of your site, you will need to change its original name to index.htm before upload the files to the relevant directory of the live server.

HOW TO GET DEMO IMAGES:

The final package that you can buy comes with placeholders due the strict copyright rules of Envato, so I cannot include images in the downloadable package regardless its licence, but I prepared some Unsplash images that you can download from here. NOTE: I totally have the right to share or redistribute those images, so don't worry about copyright issues, you can even show them on your site.

IMPORTANT NOTE!

Chrome cannot handle ajax calls locally due to its cross-origin security rule. That means the portfolio item pages, service items pages, team members will not load properly. So, how to fix it?

1) Use Firefox for local development
2) Install LAMP to your local computer for instance xampp (for windows), appserw (for windows) and mamp (for mac). I use Appserv and the ajax items load perfectly from localhost. 3) Work on your live server and update the files via ftp all the time.

It is also good to know that some browsers caches ajax contents really aggressively, so you might need to delete the cache if you want to see the changes you made to files in the ajax_contents directory.

Entrance Animations

As you can see in the demo several elements revealed with an animated effect upon scrolling down the site. You can control the animations by using the following data attributes :

data-animation - possible values: true / false
data-animation-style - The theme uses the animate.css library to show or hide elements, so please check out the possible values here.

It is important to know that none of the mentioned data attributes can work alone. You must add both of them to the element you want to animate! If you add data-animation="true" to an element and don't set up the style, the theme will do nothing when you scroll the site! So, again, please add both of the data attributes to the selected html tag, for example: 

               <div data-animation="true" data-animation-style="fadeIn">

Inside the <head>

The head tag contains all of the global information of the document that the browser need to know for instance the title, default charset, the location of the stylesheets etc. You can also specify an icon to represent your webpage on Apple devices. 

                    <head>
                    
                    <!-- Title -->
                    <title>[... The title of your website ...]</title>
                    
                    <!-- Character set -->
                    <meta charset="UTF-8">
                    
                    <!-- Configure viewport for mobile / tabled devices -->
                    <meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0">
                    
                    <!-- Define keywords for search engines -->
                    <meta name="keywords" content="[... List of keywords for search engines ...]">
                    
                    <!-- Define a description of your web page -->
                    <meta name="description" content="[... Description of your web page for search engines ...]">
                    
                    <!-- Shorcut icon -->
                    <link rel="shortcut icon" href="[... path to the favicon ...]">
                    
                    <!-- Apple Touch Icon  -->
                    <link rel="apple-touch-icon-precomposed" sizes="152x152" href="[... path to the touch icon ...]">
                    
                    <!-- Call Google fonts -->
                    <link href='https://fonts.googleapis.com/css?family=Montserrat:400,700' rel='stylesheet' type='text/css'>
                    <link href='https://fonts.googleapis.com/css?family=Lato:900,700' rel='stylesheet' type='text/css'>
                    <link href='https://fonts.googleapis.com/css?family=Droid+Serif:400,400italic,700,700italic' rel='stylesheet' type='text/css'>
                    <link href='http://fonts.googleapis.com/css?family=Quicksand:300,400,700' rel='stylesheet' type='text/css'>
                    
                    <!-- Styles -->
                    <link rel="stylesheet" type="text/css" href="style/font-awesome.css" />
                    <link rel="stylesheet" type="text/css" href="style/animate.css" />
                    <link rel="stylesheet" type="text/css" href="style/style.css" />
                    
                    </head>

Styles, meta tags and the title are mandatory, so please do not remove them! The shortcut icon and touch icon can be removed in case you don't need them.

It is also very important to know that ajax calls might not work properly on a local computer! That means the script will return with the Content cannot be loaded error message. Please use a local webserver application such as Appserv or XMAPP in order to get ajax calls work flawlessly. I'd like to note that this is not a problem with the theme, this is how AJAX works. Some browsers implement strong security measures to prevent downloaded webpages from accessing arbitrary files on the file system, in other words you cannot load anything via ajax if the url doesn't start with http:// or https://. Obviously urls like file:///E:/work/webdesign/Templates/MarkZ/Elastic/final/html/ajax_contents/services/sample_service.htm doesn't have any http:// therefore the ajax call will not work perfectly. If you install one of the mentioned server applications or upload the project to a real hosting server, the ajax calls will work perfectly.

FAQ / Troubleshooting

Portfolio items, services, team members, lightbox images etc. don't load, what to do now?

All of those sections of the theme use ajax to load item contents. Newer browsers implement strong security measures to prevent downloaded webpages from accessing arbitrary files on the file system, so they don't load urls that starts other than http:// or https://. Urls like file:///E:/work/webdesign/Templates/MarkZ/Elastic/final/html/ajax_contents/portfolio_items/slider.htm obviously don't contain any http:// therefore they will not be loaded. What is the solution? I recommend you to use a local webserver application such as Appserv or XAMPP. Both of the mentioned applications turns a selected directory into a fully working, local webserver, so you will have directories with urls like http://localhost/yourproject which is perfect for ajax.

Although Appserv and XAMPP are absolutely harmless ( turning your computer into a webserver doesn't mean that any people can reach your computer through internet! ) you might not want to tweek with those application. So, in this case you will need to upload the entire project to your hosting account and updating the files all the time via ftp. Sometimes you will need to clear the cache of your browser to see changes you made in ajax loaded contents!

You said the word "Ajax" many times. What is this at all?

Ajax (also AJAX; an acronym for Asynchronous JavaScript and XML) is a group of interrelated web development techniques used on the client-side to create asynchronous web applications. With Ajax, web applications can send data to, and retrieve data from, a server asynchronously (in the background) without interfering with the display and behavior of the existing page. More info on WikiPedia.

I changed the contents of a portfolio item, but I didn't see any changes when re-loaded the item!

This is another ajax issue! :) Browser caches ajax loaded contents really aggressively, so you need to clear cache of your browser when updated the item.

How can I activate the percantage-preloader?

                                    <body>

and this is how the active should look: 

                                    <body>

The percentage loader stuck at XY % and I can't get it work.

The script that hides the preloader is located in a $window.load() event, so it hides the overlay element only when the site is fully loaded. There must be a broken link or missing resource if the loader stuck. Deactivate the preloader by commenting out its html code and check the console for error messages.

How can I disable the entrance animations?

You have two options to disable entrance animations. If you want to disable the animation for some selected items, you just need to edit / remove the relevant data attributes of those items.

If you want to disable all the entrance effects completely, you will need to remove or comment out the line 406 from the scripts/custom.js file: 

                               /*-- Init Animation Handler --*/
                               new ENTRANCE().init();

The contact form doesn't work. What am I suppose to do?

Unfortunately there may be several causes. Some of them are easy to solve while the others are absolutely unsolvable. Let's see some possible solutions:

1) Check all settings inside the mail.php. Maybe you misspelled your email address or there is an unexpected character somewhere.

2) If the mail.php is absolutely ok, try to use the mail2.php instead. Open that file and set up the email address and other required settings. Once the mail2.php prepared, open the scripts/custom.js file and change the line 379: 

                               url: 'mail2.php',


3) If none of the tips above help, the problem will be with your server. The mail.php require the PHPMailer class. Even though the PHPMailer is a reliable and widely used php class, there are servers which don't support it. The mail2.php is created especially for this case. It uses a simple mail() function to send emails. If none of them work, there will be a server side issue. I'm sure.

4) You can still try to use smtp if your server doesn't send emails at all. You might find the following resources helpful:

Basic Example using SMTP
Basic Example using Gmail
